unix のかんたんなコマンド解説を送ります。徐々に慣れればと。

#===================================
# July 23, 2020 K.E.
# some commands of UNIX
#----------------------

◆ unix (FreeBSD, Linux, ...) の console 画面でよく使う commands

man, ls, cp, mv, less, more, pwd, cd, mkdir, rm, chown, chmod,
cal, date, time, clear, [enter]


1) ls
list directory contents.
directry にある file、directry、link などの表示

ただし、このままだと表示が簡易すぎるので、一般的には引数をつけて

% ls -lAF

などとする。引数をつけるのも面倒なので、alias 機能で、引数付きの command
にするのが一般。alias 機能の実現は、使用する shell によって、定義ファイル
が異なる。

FreeBSD の default shell は csh。その定義ファイルは .cshrc で、先の ls
の alias であれば

alias ll        ls -lAF

としておけば、あとは、ls の代わりに ll とすればよい。

2) cp
file や directry などの複写(コピー)。Win系の複写は、元ファイルなどの属性
(作成日付や許可属性)もいっしょに複写するが、UNIX系のは属性を引き継がない。
なので、日付属性などを維持したいとき(Win系のコピーと同じ状態)は

% cp -p [元] [複写]

とする。

3) mv
move の略なので移動になるが、実態は rename に近い。

% mv [元] [新しい名前]

4) less または more
text file (ASCII code) の内容を console に表示。
いまの FreeBSD では、more は less に置き代えられている。less は進化版。
ls などで表示が console 1画面内に収まらないとき、more だと一瞬で表示が流
れてまう。less だと、ひとつの画面ごとに停止し、space key 押下すると、次の
表示に進む。途中で止めたいときは、q または ctrl+c 押下。

% less [sample_file]

5) pwd
return working directory name. 自分が居る場所の表示

6) cd
change directry. 居場所の変更。
UNIX における HDD、SSD の領域は、以下のよう。

  bin              <DIR>  システムの基本 command
  boot            <DIR>  起動関連 command
  compat          <DIR>  他システム用の command
  dev              <DIR>  device との接続
  etc              <DIR>  システムの基本設定
  home            <DIR>  利用者が使える領域        <- (A)
  lib              <DIR>  システムのlibrary
  libexec          <DIR>  同上(実行属性があるもの)
  net              <DIR>  ネット環境用のシステム command
  root            <DIR>  root が使用する領域
  sbin            <DIR>  システムの基本 command
  tmp              <DIR>  システムの一時保存領域
  usr              <DIR>  利用者がインストールした commands
  var              <DIR>  システムの log関連領域

このうち、利用者が自由に使えるのは、(A) の配下のみ。通常は

/home/user_name

が割り当てられ、その配下のみ、自由に使えるので、使いやすいように directry
を作成して、各 directry に作業の意味を割り当てることが多い。永村の場合は

/home/emura/配下

  Downloads    <DIR>    DLする際の一時的な置き場所
  bin          <DIR>    自分でつくった commands
  i7          <DIR> --- 00crag                  <DIR> ファーナム先生関連
                        00e2ko                  <DIR> 自宅関連
                        00misc                  <DIR> その他
                        00wk                    <DIR> 一時的な置き場所
                        00yuan                  <DIR> 袁先生関連
                        2020                    <DIR> 2020年の仕事関連
                        22.misc                  <DIR> 一時的な置き場所
                        Asahi                    <DIR> 新聞記事など
                        Audio                    <DIR> 音楽関連
                        PT.city                  <DIR> 過去の仕事関連
                        School                  <DIR> 教務関連
                        Soc                      <DIR> 学会関連
                        amedas                  <DIR> AMeDAS関連
                        archiv                  <DIR> 学術関連
                        archiv.pcs              <DIR> PC機材関連
                        email                    <DIR> メール
                        pool                    <DIR> その他保存すべきもの
  ref          <DIR>    Tex や特殊なfont などの置き場所
  tmp          <DIR>    一時的な置き場所
  usage        <DIR>    自分で調べた FreeBSD の使い方や command などの仕様

こんな具合になっている。

自分が console で、どこに居るのか(console に表示されている場所)を pwd で
確認したあと、行きたい場所に行くときに使う command が cd になる。

% cd [行きたい directry]

7) mkdir
make directry. directry の作成

8) rm
remove. file、directry の削除

% rm [file]

% rm -rf [directry]

9) chown
change file owner and group. 所有属性の変更

UNIX では、ファイルは、作成者がだれか、それはどのグループか、がわかるよう
になっている。たとえば

% ll s.f95
-rw-r--r--  1 emura  adm  267 Jul 16 17:34 s.f95

所有者：emura
グループ：adm
作成日：July 16, 2020

user の emura の基本属性は、adm グループだが、ほかにも複数のグループに属
しているので、ファイルによっては

% ll ./Download
-rw-r--r--  1 emura  wheel    5196 Jul  7 16:25 hf

となって、wheel というグループのファイルも持っている。これを、基本の adm
にしたければ

% group emura:adm ./Download/hf

とすると

% ll ./Download
-rw-r--r--  1 emura  adm    5196 Jul  7 16:25 hf

となる。

システム側のファイルの大半は、root(super user)が所有しているので、システ
ム commands は、たいてい、root属性がついている。rootの基本グループ名が、
wheel である。

% ll /usr/local/bin/firefox
lrwxr-xr-x  1 root  wheel  30 Jul 10 19:57 /usr/local/bin/firefox@ -> /usr/local/lib/firefox/firefox

10) chmod
change file modes. ファイル属性の変更

ls でファイル、ディレクトリ、リンクなどを表示させたとき、左端に表示される

% ll s.f95
-rw-r--r--  1 emura  adm  267 Jul 16 17:34 s.f95
~~~~~~~~~~
この表示が mode と呼ばれる属性で、ファイルであれば、読み書き、ディレクトリ
であれば、そこに入れるか否かを表わし、実態のあるものか否かも表示できる。
# 実態がなくとも、link を張って、見かけ上、実態とは異なる名前のファイルを
# 作成することもできる。

詳細は、ググって調べること。
